# 变更方案模板

复制本模板建立具体计划。方案在用户明确确认前保持“待确认”，不得提前修改 tracked 文件。

## 状态

- 状态：草案 / 待确认 / 已批准 / 实施中 / 已验证 / 已完成 / 已挂起
- 提出日期：YYYY-MM-DD
- 基线 commit：`<commit>`
- 用户确认：未确认 / `<确认日期与范围>`

## 当前行为与证据

说明可复现现象、相关文件、命令输出、版本和环境限制。区分仓库事实、本机状态和推测。
